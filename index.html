LambdaCon subjects

Sessions will show how to leverage the power of functional programming languages such as Scala, Haskell, Clojure, Erlang, F#...
Nonetheless the conference aims to instill the functional mindset in attendees so that they can use it as a tool even when programming with an object oriented language

Attendees will learn basic and advanced concepts and how to use FP in their everyday life

  • immutability, higher order functions, currying, continuation-passing, pattern matching, monads
  • DSLs, domain modeling, actor model, machine learning
  • multithreading, non-blocking IO, message passing, message queues, microservices, HTTP-based APIs, web applications, high availability, continuous delivery
Conference structure

The program runs across multiple tracks
Sessions may be given in the form of a talk or a workshop

TALK
A presentation conducted in a lecture format and focused on a single topic: a language, technique or tool as well as an experience report
Primarly led by the speaker, a talk should also include some discussion

WORKSHOP
A hands-on session, focused around a specific language, technique, tool or issue, in which participants learn linking theory and practice by coding along with the speaker and experimenting

Learn more on LambdaCon

A conference for people eager to know more about functional programming, combining talks with hands-on sessions to take your skills to the next level

LambdaCon is focused on learning the functional approach, understanding in which scenarios it is better to use FP, discovering what can be done with current languages and technologies and exploring current best practices

Network with other funatics, learn something new, share your knowledge and experiences and have a lot of fun

Opening keynote - Fun with categories

A category is an embarrassingly simple yet powerful concept. It's behind Unix pipes, simple arithmetic, programming types, functions, state and side effects. I will show you how categories can be used to design generic C++ libraries.

Why bother with FP?

Instead of just bashing other paradigms I'll provide real arguments for FP, backed.
I’ll talk about:
  • complexity, accidental and essential
  • state manipulation / mutation
  • pure OOP model (Smalltalk, though model is quite language agnostic)

Tomek Borek

F# in the real world

F# is an incredibly versatile and expressive language and enables you to solve a wide array of problems succinctly and efficiently.
This talk illustrates, with real-world examples, how the Gamesys social team is using F# to build the backend services that support around 1 million Daily Active Users (DAU) and 250 million daily requests across its social games on both mobile and web platforms.
Talk objectives:
  • Demonstrate the use of F# to write clean, concise game logic as well as infrastructure code, the same techniques and concepts can be easily applied to other domains
  • Demonstrate how F# can help tackle recurring business problems such as efficiency, correctness, complexity and time to market

Yan Cui

Using Haskell professionally

Studying FP during free time is a lot of fun, but most people are reluctant in making an effort learning a functional language because they think it would be impractical to use in the everyday job industry.
This talk is going to show that not only is it possible to use Haskell to earn a living, but that the benefits from immutability, purity and conciseness that Haskell offers are the perfect ingredients to make a business succeeds.

Alfredo Di Napoli

From imperative to functional APIs

Libraries and APIs often shape how our program looks like. Unfortunately, we often have to deal with imperfect libraries that have a brittle API, partly due to a lack of care, partly to language restrictions.
In this talk we will explore how we can use features of OCaml, a functional programming language, to improve an interface provided by a C library.

Marek Kubica

Clojure redeployed

At stylefruits we've been using Clojure on production since late 2012. We've made a lot of mistakes and we've learned a lot.
This talk is dedicated to the newest addition to our infrastructure: a Clojure REST-inspired API.
Taught by our previous mistakes in the realm of deployment and scalability we've set our requirements high: we're aiming for continuous delivery, high availability, dynamic scaling and pauseless deployments under load.
Aside from deployment matters, Clojure proves to be an excellent choice to build a modular and stateless service.
In this talk we're going to share our experiences, both good and bad, gained while prototyping, deploying and maintaining our newest project.

Jan Stępień
